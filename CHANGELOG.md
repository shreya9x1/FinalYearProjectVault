### abhishek commit for SAR
This commit adds a Jupyter notebook file containing a complete implementation of Synthetic Aperture Radar (SAR) machine learning components for MSTAR dataset classification. The notebook includes RCNNet backbone, PatchWeightMCL head, DMM module, strategic imbalance handling, and open-set recognition for few-shot learning tasks. The code runs training and evaluation pipelines achieving ~99.6% known accuracy and 51.6% overall open-set accuracy.

Added complete submission for the Joint Super-Resolution and Multi-Label Classification for Remote Sensing Imagery final year project by Shreya Pragna. This includes a comprehensive technical report, presentation slides, and two Jupyter notebooks containing the full data preparation pipeline and integrated model implementation. The project demonstrates a novel approach to jointly optimize satellite image enhancement and semantic land-cover classification using FLAIR-2 dataset with a three-stage progressive training strategy combining CVAE-based super-resolution, ResNet50-based multi-label classification, and Graph Neural Networks for spatial and label reasoning.

Initial creation of the FinalYearProjectVault repository. This marks the establishment of a centralized system for managing final year student projects under the guidance of Shounak Chakraborty.

#### Project Overview

FinalYearProjectVault is designed as a comprehensive repository to organize and store all deliverables related to final year student projects. The vault provides a structured approach to managing diverse project artifacts including detailed reports, presentation materials (PowerPoint slides and presentations), source code implementations, and other project-related assets.

#### Key Features

- **Centralized Management**: Single repository for all student project submissions
- **Organized Structure**: Standardized folder hierarchy for reports, presentations, code, and additional materials
- **Version Control**: Git-based tracking of project evolution and changes
- **Scalability**: Flexible structure to accommodate multiple concurrent projects
- **Documentation**: Comprehensive README with submission guidelines and repository structure

#### Repository Structure

The repository establishes a framework for organizing student submissions with the following key directories:

- **Project Folders**: Individual directories for each student project
- **reports/**: Documentation and project reports
- **presentations/**: PowerPoint presentations and slide decks
- **code/**: Source code and implementation files
- **Supporting Materials**: Space for additional project assets and resources

#### Initial Files

- **README.md**: Comprehensive documentation including repository overview, structure guidelines, submission instructions, and contributor guidelines

#### Governance

All projects are overseen by faculty advisor Shounak Chakraborty, ensuring academic quality and adherence to institutional standards.

#### Getting Started

Students can clone the repository and begin organizing their project submissions following the established structure and guidelines. The repository provides clear instructions for contributing while maintaining proper academic integrity and documentation standards.
